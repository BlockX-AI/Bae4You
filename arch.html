§03
System Architecture

Three architectural layers, clearly bounded. The rule: economic truth on-chain, everything else off-chain, UX invisible to users. The Graph sits as the read bridge between layers.

Layer A — Client
React Native App + Next.js Web
Embedded wallet auto-created on signup. All blockchain interactions abstracted — user sees "Buy Pet" not "Sign Transaction". Gas sponsored via Paymaster. SIWE (Sign-In with Ethereum) replaces passwords silently.
Layer B — Off-Chain Backend
Fastify API + Socket.io + EIP-712 Signer
Handles all social features (chat, matching, profiles), EIP-712 signed bonus claims, notification dispatch, ranking computation via The Graph data, and fiat on-ramp orchestration.
Layer C — On-Chain Protocol
4 Contracts on Base L2
PetsRegistry (SFT mint), PetsMarket (buy/sell/10% rule), PetsCash (ERC-20), PetsRanking (badge SFTs). All economic state lives here. Immutable. Auditable. Trustless.

§06
Smart Contract Architecture
Chain
Base L2 (Ethereum rollup, Coinbase-backed). EVM-compatible. Near-zero gas. Existing SocialFi tooling and user community. Friend.tech's home chain.
Security Model
Multisig treasury (3-of-5). Pausable contracts (admin emergency). OpenZeppelin base contracts. Mandatory two independent audits + Tenderly monitoring before mainnet.
PetsRegistry.sol
ERC-1155
Mints a unique SFT for every new user. Sets initial market price. Maintains token ID → wallet address mapping. Manages ghost status on deactivation.
mintProfile()
Called on user signup. Mints SFT, sets starting price, emits ProfileMinted event.
ghostProfile()
On deactivation (user-signed). Freezes price, sets status = ghost, disables transfers.
getTokenByAddress()
Returns token ID for a given wallet address.
PetsMarket.sol
Core Trading
The heart of the protocol. Enforces 10% price invariant. Handles 50/50 profit split. Manages locks. Collects 2.5% platform fee to treasury multisig.
buy(tokenId)
Core trade function. Checks lock + balance, executes price × 1.1 increase, splits profit 50/50, emits PetPurchased. Uses basis points math throughout.
lockPet(tokenId, duration)
Owner locks a pet for a set duration. Auto-expires at block.timestamp. No manual unlock needed.
buyAgain(tokenId)
Re-acquire an owned pet at incremental cost (price diff only). Both parties must be ranked.
giftCash(tokenId, amount)
Ranked owner transfers Pets Cash to a pet. Max 10 per 24h window. Tracked by timestamp mapping.
PetsCash.sol
ERC-20
In-game currency. Minted via signed bonus claims. Transferred in all trade settlements. Bridgeable to Bae4U Gold (premium). Burnable for token sinks (premium features, tournament entry).
claimBonus(sig, timestamp)
User submits EIP-712 signed claim. Contract verifies signature and 4-hour cooldown, then mints bonus amount. Backend signer never holds owner keys.
mintFromMarket()
Called only by PetsMarket — distributes profit splits on trade settlement.
burn(amount)
Burns tokens from caller's balance. Used for premium feature purchases (token sink mechanic).
convertToGold(amount)
Burns Pets Cash, records Gold credit off-chain. One-way conversion for premium acceleration.
PetsRanking.sol
ERC-1155 Badges
Issues badge SFTs (Bronze → Master) when users cross asset thresholds. Weekly/monthly resets triggered by Chainlink Automation. Verifies signed proofs from backend before minting.
issueBadge(addr, tier, proof)
Verifies backend-signed ranking proof, then mints badge SFT. Badges are on-chain credentials — owned, not just displayed.
weeklyReset()
Called by Chainlink Automation every Monday 00:00 UTC. Snapshots current standings, triggers badge issuance queue.
getActiveBadge(address)
Returns current highest badge tier for a given wallet.

§08
Protocol Layer — Web3 Positioning
What makes this a protocol, not an app
A protocol is composable. Any developer can build on top of PetsMarket's buy() function. Any marketplace can list Pets Cash. Any wallet can hold badge SFTs. Bae4U the app is one interface to the Pets Protocol — just like Uniswap the interface sits on top of the Uniswap Protocol. The protocol persists even if Bae4U the company disappears.
Revenue model at protocol level
2.5% platform fee on every pet transaction (flows to multisig treasury). 5% royalty on secondary SFT pass sales. Premium features burn Pets Cash (token sink). Fiat on-ramp spread (1–2% on Transak embedded). Creator revenue share — 85% to creator, 15% to protocol on pass sales.
Composability
Open protocol contracts
Any dApp can integrate PetsMarket.sol. Third-party marketplaces can list SFT profiles. Badge credentials can be used as access passes in any token-gated system. Protocol-level openness drives ecosystem growth.
Identity Moat
On-chain reputation layer
A user's SFT token ID + transaction history + badge credentials form an on-chain reputation that follows them across Web3. This is the identity primitive no centralized dating app can replicate. Verified humans with provable social history.
Token Economics
Designed for longevity
Six minting sources (controlled inflation) + multiple burn sinks (weekly tournament entry, premium badges, boost features). Deflationary pressure increases with platform scale. Tokenomics designed so Cash has value independent of ETH price.

§10
Risk Register
Technical Risks
Critical — Smart contract exploit
Re-entrancy or precision error in buy(). Mitigation: CEI pattern, two audits, basis points math, 100% test coverage on profit split before mainnet.
Critical — Rounding error at scale
Integer division in 50/50 split leaks wei at scale. Mitigation: document rounding policy, always use × 11000 / 10000, excess wei goes to pet (documented).
High — Gas UX friction
Any visible gas prompt kills mainstream adoption. Mitigation: Paymaster contract sponsors all gas, embedded wallet, never surface "approve transaction" to users.
High — Bot front-running
Bots buy new low-priced accounts before humans. Mitigation: first-owner claim cooldown, rate limiting on-chain, off-chain bot detection layer.
Product + Market Risks
Critical — Token crash kills engagement
Pets Cash price crash → users feel poor → churn. Mitigation: dating utility independent of token price. Token sinks. Dating-first onboarding where crypto is invisible.
High — Whale dominance kills new users
Top 0.1% own all rankings (Friend.tech precedent). Mitigation: country rankings, activity bracket tiers, weekly resets ensure fresh competition windows.
High — Dating safety + harassment
Pets mechanic can feel dehumanising without framing. Mitigation: identity verification layer, opt-out of Pets discovery, strong community guidelines, moderation queue.
Medium — Regulatory (securities)
Pets Cash may be classified as a security in US/EU. Mitigation: legal opinion before mainnet, utility token framing, no speculative marketing, Howey test analysis.
